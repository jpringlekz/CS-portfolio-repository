# DriverPass System Analysis and Design Project

## Overview

This repository contains my coursework for the DriverPass system analysis and design project completed in SNHU CS 255. The project demonstrates my ability to gather business requirements, analyze user needs, and design a software system using UML diagrams and technical planning.

Included in this repository are the Business Requirements Document and the System Design Document for the DriverPass client project.

## Project Summary

The client for this project was DriverPass, a company that wanted a system to help students prepare for DMV driving tests and schedule on-road driving lessons. The system needed to allow customers to create accounts, take online practice tests, schedule lessons, and allow employees to manage users, reservations, and services.

The final system design included support for multiple user roles such as customers, employees, administrators, and drivers. It also used a cloud-based environment for accessibility and scalability. :contentReference[oaicite:0]{index=0}

## What I Did Well

One area I performed well in was translating customer needs into clear system requirements. I also created UML diagrams that modeled how the system would function, including use case, activity, sequence, and class diagrams. These helped organize the design and communicate ideas clearly.

## What I Would Improve

If I could revise one part of this project, I would improve the database and class design by adding more detailed relationships between reservations, payments, notifications, and reporting tools. I would also expand the security section with stronger authentication and account recovery features.

## Understanding User Needs

I interpreted the user’s needs by reviewing the DriverPass interview transcript and identifying the problems they wanted to solve. I then converted those needs into features such as scheduling lessons, taking practice tests, managing accounts, and storing customer data securely.

It is important to consider user needs because successful software must solve real problems while being easy to use, secure, and reliable.

## My Software Design Approach

My approach to software design begins with understanding business goals and gathering requirements. Then I break the system into smaller components and model the solution with diagrams such as use case, activity, sequence, and class diagrams.

In the future, I would continue using planning, prototyping, stakeholder feedback, and iterative improvements when designing systems.
